ROOT Web Application

Overview

This sample application demonstrates how to write a default home applicetion to work on "ROOT".


Understanding the Sample

The ROOT web application was written to support binding different applications to specific local ports. The binding rules defined in the web.xml of ROOT web application.

Following is an example of the ROOT web application wen.xml: 

      <servlet>
        <servlet-name>WebForwarder</servlet-name>
        <servlet-class>com.micromethod.sipmethod.sample.webroot.WebForwarder</servlet-class>
        <init-param>
          <param-name>8080</param-name>
          <param-value>http://www.voxeo.com</param-value>
        </init-param>

        <init-param>
          <param-name>8081</param-name>
          <param-value>/echo</param-value>
        </init-param>

        <load-on-startup>1</load-on-startup>
      </servlet>

      <servlet-mapping>
        <servlet-name>WebForwarder</servlet-name>
        <url-pattern>/*</url-pattern>
      </servlet-mapping>
In this example, the http://serverhost:8080 accessing will be redirected to http://www.voxeo.com, the http://serverhost:8081 accessing will be redirected to http://serverhost:8081/echo.
Make sure the ports were configured in sipmethod.xml:
<NetworkAccessPoint transport="http" address="0.0.0.0" port="8080"/>
<NetworkAccessPoint transport="http" address="0.0.0.0" port="8081"/>


Running the Sample

The top level directories and files in this sample are:

docs/ the documentation about this sample
ROOT.sar the application archive file (SAR)
ROOT/ the exploded application directory

Please follow the following steps to deploy, configure and run this sample:

  1. Copy ROOT directory into apps directory.

  2. Configure the initialization parameters in apps/ROOT/WEB-INF/web.xml.
