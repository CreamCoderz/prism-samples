XMPP client

Overview

This sample application demonstrates how to implement a XMPP client using XMPP servlet technology.


Understanding the Sample

There are two servlets in this sample. One is a HTTP servlet, named XMPPClientHTTPServlet. The XMPPClientHTTPServlet expose a HTTP URL, http://localhost:8080/xmppclient/XMPPClient. When you access this URL, the servlet will use XMPP servlet API to initiate a session to gtalk using account testxmppclient@gmail.com. After the session has been created(i.e logon to gtalk), the servlet will send a message to account testxmppclient2@gmail.com. So if testxmppclient@gmail.com is online, it will receive a message "hello, this is XMPP servlet client." The second servlet is XMPPClientReceiveBackMessageServlet, it is a XMPP servlet, used to accomplish the following negotiation process after the XMPPClientHTTPServlet initiate a session to gtalk, it is also used to receive the back message and just echo it.


Running the Sample

The top level directories and files in this sample are:

docs/ the documentation about this sample
xmppclient.sar the application archive file (SAR)
xmppclient/ the exploded application directory

Please follow the following steps to deploy, configure and run this sample:

  1. Copy xmppclient directory (or xmppclient.sar) into apps directory.

  2. Start SIP servlet container (SIPMethod Application Server) as user guide.

  3. logon to gtalk using account testxmppclient2@gmail.com(password testxmppclient2_1).

  4. Access the URL http://localhost:8080/xmppclient/XMPPClient, you will see a response page that says "Sent message to: testxmppclient2@gmail.com". The account testxmppclient2@gmail.com will receive a message "hello, this is XMPP servlet client." from testxmppclient@gmail.com. When you talk with it, it will echo your message.
