Registrar Sample

Overview

This sample demonstrates a simple registrar. SIP client can register to this sample, then other sample can query register information from this sample.


Understanding the Sample

The SIP servlet - Registrar - demonstrates how a simple register server handles SIP REGISTER. It maintains a list of registered user agent clients (e.g. X-Lite) and save it to the ServletContext.

The HTTP part - QueryAddressServlet - demonstrates how HTTP servlet and SIP servlet can exchange information via ServletContext. QueryAddressServlet shares the same servlet contexts as Registrar servlet so it can retrieves the address-of-record and contact address mapping maintained by EchoServlet. When AddressServlet receives HTTP GET or POST request, it builds a HTML page to display the list of the mapping.

AddressManagerImpl implements AddressManager to invoke SOAP message request, it also shares the same servlet contexts as Registrar servlet, When web service receives "list" or "remove" request, it will display the list of the mapping or remove input uri's mapping.

For more details, please see source codes.


Running the Sample

Please follow the steps below to run a X-Lite with this sample:

  1. Copy registrar directory (or registrar.sar) into $INSTALL_DIR/apps directory. Start Prism Application Server if you haven't.

  2. Startup and Configure your X-Lite (or other compatible SIP Clients).

    • Open X-Lite 3.0 or later.
    • On the display board, write click and select "SIP Acount Settings...".


    • On the SIP Accounts panel, click Add....


    • Type the User Details and Domain Proxy as follow. In fact, you can use any account info you like.
    • Then click OK.


  3. Register your X-Lite to this application.

    • Start SIP servlet container (Prism Application Server) as user guide.
    • Start X-Lite,and it will register to the container.


  4. By now, Sip client registered to this sample. Then other sample can get this register information by using a HTTP request to this sample.The default query URL is "http://localhost:8080/registrar/query". You can also use browser to access this URL to get register information.

  5. Use web browser to view the Registered Addresses. Start the web browser and visit http://localhost:8080/registrar/query.

  6. Running testclient script to list or remove registered addresses by web service.

    For example: 

    >testclient -lhttp://localhost:8080/registrar/services/manager list
>testclient remove "sip:test@registrar.sample.prism.voxeo.com"
