Minimal Conference Sample

Overview

This sample application demonstrates how to write a SIP servlet using JSR-309 API to work with media server.

MinimalConferenceServlet is such a service, that all callers are conferenced together, a single conference for all, everybody can talk/listen.


Understanding the Sample

Once MinimalConferenceServlet is initialized, it creates a MediaMixer, indicating the single conference.

Upon the receipt of the INVITE message, MinimalConferenceServlet will create a NetworkConnection, and join it to the MediaMixer in DUPLEX direction. Then request the media server to process the given SDP offer from the remote User Agent (e.g. XLite). Once SDP negotiation is complete, MinimalConferenceServlet will send back SIP/200OK to the remote User Agent. As a result, the remote User Agent can talk to/listen from the conference.

When MinimalConferenceServlet receives BYE message, it will release the NetworkConnection. Then send back SIP/200OK to the remote User Agent.

For more details, please see source codes.

The Media Server location can be configured via sipmethod.xml which can be found in folder $SM_HOME/conf, as shown in the following.


   <MediaControl MEDIA_SERVER_URI="mrcp://127.0.0.1:9974" VCM_SERVER_URI="http://127.0.0.1:9995;virtualPlatformId=Default"
        VXML_BROWSER_URI="sip://127.0.0.1:5066;transport=tcp"/>

Attribute Description
MEDIA_SERVER_URI URI of Media Server
VCM_SERVER_URI URI of Voxeo Conference Manager
VXML_BROWSER_URI URI of Voice XML Browser

Running the Sample

The top level directories and files in this sample are:

docs/ the documentation about this sample
miniconf.sar the application archive file (SAR)
miniconf/ the exploded application directory

Please follow the following steps to deploy, configure and run this sample:

  1. Start Voxeo Prism Media Server if you haven't.

  2. Copy miniconf directory (or miniconf.sar) into apps directory.

  3. Use UA(e.g. X-Lite) to make the call to server. You should configure the domain of the callee as miniconf.sample.prism.voxeo.com. You should configure your XLite to send outbound via proxy. By default, the proxy Address of your XLite configuration should be "127.0.0.1:5060" if the User Agent is on the same machine as SIPMethod Application Server, and the SIP listening port of SIPMethod server is 5060.
