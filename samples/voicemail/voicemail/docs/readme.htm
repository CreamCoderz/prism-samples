Voicemail Sample

Overview

This sample application demonstrates how to write a SIP servlet using JSR-309 API to work with media server.

VoiceMailServlet is a very simple voicemail application that answers incoming calls. The caller can send out a voice mail or listen to his received messages.


Understanding the Sample

Upon the receipt of the INVITE, VoiceMailServlet will create a NetworkConnection and a MediaGroup joining them together in DUPLEX direction, and request the media server to process the given SDP offer from the remote User Agent (e.g. XLite). Once SDP negotiation is complete, VoiceMailServlet will send back SIP/200OK to the remote User Agent.

When VoiceMailServlet receives ACK message, it will start to play the welcome message, and wait for the caller to input signal "*" to start the voicemail service. Once the service is started, the caller will hear a message to tell him to input his user id. On receipt of the user id, VoiceMailServlet will play back the voicemail menu to the caller, and the caller could decide whether to send out a voice mail, listen to his received messages or quit the service.

For more details, please see source codes.

The Media Server location can be configured via sipmethod.xml which can be found in folder $SM_HOME/conf, as shown in the following.


   <MediaControl MEDIA_SERVER_URI="mrcp://127.0.0.1:9974" VCM_SERVER_URI="http://127.0.0.1:9995;virtualPlatformId=Default"
        VXML_BROWSER_URI="sip://127.0.0.1:5060;transport=tcp"/>

Attribute Description
MEDIA_SERVER_URI URI of Media Server
VCM_SERVER_URI URI of Voxeo Conference Manager
VXML_BROWSER_URI URI of Voice XML Browser

Running the Sample

The top level directories and files in this sample are:

docs/ the documentation about this sample
voicemail.sar the application archive file (SAR)
voicemail/ the exploded application directory

Please follow the following steps to deploy, configure and run this sample:

  1. Start Voxeo Prism Media Server if you haven't.

  2. Copy voicemail directory (or voicemail.sar) into apps directory.

  3. Use UA(e.g. X-Lite) to make the call to server. You should configure the domain of the callee as voicemail.sample.prism.voxeo.com. You should configure your XLite to send outbound via proxy. By default, the proxy Address of your XLite configuration should be "127.0.0.1:5060" if the User Agent is on the same machine as SIPMethod Application Server, and the SIP listening port of SIPMethod server is 5060.
