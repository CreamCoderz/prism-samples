Find-Me Sample

Overview

This sample application demonstrates how to write a Find-Me application using built-in SIP proxy function.

The RFC 3261 defines a proxy as a logical entity that route SIP requests to user agent servers and SIP responses to user agent clients. A request may traverse several proxies on its way to a UAS. Each will make routing decisions, modifying the request before forwarding it to the next element. Responses will route through the same set of proxies traversed by the request in the reverse order.

Find-Me application allows the caller to track down the callee with multiple numbers. Find-Me application can try each number either sequentially or parallel until one of them is reached or all of them are failed. Trying each number in parallel is sometimes called Single Line Extension application.


Understanding the Sample

FindMeServlet simply forwards INVITE and subsequent message to a set of callee's numbers configured via sip.xml. In reality, it probably will look up the location service or other data sources to dynamically locate the callee's numbers. A "findme" parameter toggles between sequential search and parallel search of the multiple numbers.

Running the Sample

The top level directories and files in this sample are:

docs/ the documentation about this sample
findme.sar the application archive file (SAR)
findme/ the exploded application directory

Please follow the following steps to deploy, configure and run this sample:

  1. Copy findme directory (or findme.sar) into $INSTALL_DIR/apps directory.

  2. Configure a set of SIP URI as multiple callee's number in the "contacts" parameters for FindMeServlet in apps/findme/WEB-INF/sip.xml. Each URI should be separated by comma.

  3. Configure the "findme" parameter for FindMeServlet in apps/findme/WEB-INF/sip.xml. "true" means searching sequentially.

  4. Make the Caller User Agent to call Callee User Agent at the server. You should configure the user name of the callee account with findme.
