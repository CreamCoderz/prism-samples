Click-to-Dial Sample

Overview

This sample application demonstrates how to implement a web-based click-to-dial with a converged HTTP/SIP application.

Click-to-Dial establishes a call between participants using web interface or web services. It simplifies dialing in that callers do not have to dial lengthy addresses and they keep their phonebooks separately from SIP phones.

In its simplest form a user has a web page where he can enter SIP addresses of two users and SIP user agents of those two users get connected.

Another form is to have a web service to allow other applications or users to enter SIP addresses of the two users and make them connected.

Understanding the Sample

This Click-to-Dial sample has a simple Click-to-Dial servlet, which simply creates and sends INVITE to UAs and copy SDP content from response to ACK.

A user can input two URIs and click OK button on the HTML page generated by a HTTP servlet to trigger a SIP servlet to call and connect these two URIs. The SIP Servlet creates two INVITEs (no SDP) base on URIs and send to targets respectively. For each OK repsonse from the UA, the SIP servlet creates ACK message and copy SDP content base to the OK repsonse from other UA. RTP sessions be established between participants thereafter.

A user can also input two URIs via the Web Service client to trigger the Service Endpoint on the server to call and connect these two URIs.

For more details, please see source codes.


Running the Sample

The top level directories and files in this sample are:

docs/ the documentation about this sample
clicktodial.sar the application archive file (SAR)
clicktodial/ the exploded application directory
client/ the web service client application directory

Please follow the following steps to deploy, configure and run this sample:

  1. Copy clicktodial directory (or clicktodial.sar) into $INSTALL_DIR/apps directory.

  2. Configure the servlet-mapping, URL-mapping and web-service for Click-to-Dial server in apps/clicktodial/WEB-INF/sip.xml, apps/clicktodial/WEB-INF/web.xml and apps/clicktodial/WEB-INF/soap.xml

  3. Configure XLite and register to this application, You should configure the domain of the client account as clicktodial.sample.prism.voxeo.com.

  4. Please make sure the composition option is disable, WebConsole -> SIP -> CompositionLevel -> disable.

  5. Browsing the web interface and make call.

  6. Running testclient script to make call by web service.

    For example:>testclient "sip:user1@clicktodial.sample.prism.voxeo.com" "sip:user2@clicktodial.sample.prism.voxeo.com"
